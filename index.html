Frequently Asked Questions

1. The difference between var, let, and const
Use const by default for variables that won’t change.
Use let for variables that need to be reassigned.
Avoid var in modern JavaScript to prevent scope-related issues.
2. The difference between map(), forEach(), and filter()
Use map() when you need a new array with transformed values.
Use forEach() when you want to perform an action for each element without creating a new array.
Use filter() when you need a new array containing only elements that meet a specific condition.
3. Explain arrow functions and how they are different from regular functions
1. This Binding:
Regular functions: this depends on how the function is called.
Arrow functions: this is inherited from the surrounding scope (lexical this).
2. Arguments Object:
Regular functions: Have their own arguments object.
Arrow functions: Do not have arguments; use rest parameters (...args) instead.
3. Constructor Usage:
Regular functions: Can be used as constructors with new.
Arrow functions: Cannot be used as constructors.
4. Syntax:
Arrow functions are shorter and more concise, especially for single-expression functions.
4. How JavaScript Promises work?
A Promise in JavaScript represents the eventual completion (or failure) of an asynchronous operation and its resulting value. It is a way to handle asynchronous code more cleanly and avoid "callback hell."
5. How closures work in JavaScript?
A closure is a function that retains access to its lexical scope (variables from its outer function) even after the outer function has finished executing. This allows the inner function to "remember" and access those variables.
